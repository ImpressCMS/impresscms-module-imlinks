Conversion from WF-Links 1.06 --> imLinks


Instructions for converting from WF-Links 1.06 to imLinks.
If you want to do a fresh install of imLinks please select 'Install'.


Remember: It is always a good idea to make a database backup before installing any modules.

 

1. Make a backup

Backup the WF-Links tables from your database.

2. Upload the module to your website

Upload the 'modules/imlinks' folder to your {icms-rootdirectory}.
You don't have to upload the folder 'uploads/images' because imLinks will use the one from WF-Links.
If you wish to rename the install folder of imLinks than you have to do this before you install the module.

3. Change and verify folder permissions

CHMOD the following folders to 777:

{icms-rootdirectory}/uploads/images
{icms-rootdirectory}/uploads/images/category
{icms-rootdirectory}/uploads/images/category/thumbs
{icms-rootdirectory}/uploads/images/flags
{icms-rootdirectory}/uploads/images/flags/flags_small
{icms-rootdirectory}/uploads/images/screenshots
{icms-rootdirectory}/uploads/images/screenshots/thumbs
{icms-rootdirectory}/uploads/images/thumbs

4. Install the module

Login as administrator and enter the ImpressCMS Control Panel (administration). Select Modules and install imLinks.

5. Start the conversion script

Point your browser to {icms-rootdirectory}/modules/imlinks/update.php and execute the update script. Follow the instructions provided during the upgrade procedure. The script will try to determine if you have the correct version of WF-Links installed and will try to update it.

6. Update the module

Return to Control Panel --> Modules and update imLinks.

7. Configure the module

Most importent steps now will be to setup the group permissions for the module and its blocks via Control Panel --> Groups.

8. Removing conversing script

After a successfull conversion you should remove the file update.php and the folder {icms_rootdirectory}/update and its contents from your server.

9. Restore or remove WF-Links

You can continue using WF-Links after the upgrade. It is possible to run imLinks and WF-Links at the same time, where both module can share the folder {icms_rootdirectory}/uploads/images. If you don't want to use WF-Links anymore deactivate the old module and uninstall it.
